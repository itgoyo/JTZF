# 🔄 替换规则配置指南

## 问题：为什么替换规则不生效？

### 常见原因
1. ❌ 替换规则只添加到了部分规则
2. ❌ 替换模式未开启
3. ❌ 在错误的规则上添加了替换规则
4. ❌ 规则ID不匹配

---

## ✅ 正确配置步骤

### 第一步：查看所有规则
```bash
/list_rule
```

**输出示例：**
```
规则 ID: 1
  源: jiaotest0 (3119412600)
  目标: jiaotest (3187300742)

规则 ID: 2  
  源: jiaotest0 (3119412600)
  目标: 机器人频道 (7554364416)
```

### 第二步：确认当前操作的规则
```bash
/switch
```

**输出示例：**
```
✓ 来自: jiaotest0  ← 这是当前规则
  来自: jiaotest0
```

### 第三步：为当前规则添加替换规则
```bash
# 添加链接替换
/replace www.baidu.com www.google.com

# 添加文字替换
/replace 广告 公告

# 验证是否添加成功
/list_replace
```

### 第四步：开启替换模式
```bash
/settings
# 点击按钮开启"替换模式"开关
```

### 第五步：复制到其他规则（如果需要）

**方法1：批量添加（推荐）**
```bash
/replace_all www.baidu.com www.google.com
```
这会为所有规则添加此替换规则

**方法2：完整复制规则**
```bash
# 将规则2的所有设置复制到规则1
/copy_rule 2 1
```

**方法3：只复制替换规则**
```bash
# 切换到目标规则
/switch
# 选择规则1

# 复制规则2的替换规则
/copy_replace 2
```

## 🎯 实战示例

### 场景：同一源频道转发到两个目标频道，都要替换链接

**当前状态：**
- 规则1: jiaotest0 → jiaotest（目标频道1）
- 规则2: jiaotest0 → 机器人频道（目标频道2）

**目标：**
两个目标频道都要将 `www.baidu.com` 替换为 `www.google.com`

**解决方案1：使用批量命令**
```bash
# 1. 在任意频道执行（会应用到所有规则）
/replace_all www.baidu.com www.google.com

# 2. 验证（每个规则都检查一次）
/settings 1  # 查看规则1，确认替换规则和替换模式开启
/settings 2  # 查看规则2，确认替换规则和替换模式开启
```

**解决方案2：逐个添加**
```bash
# 1. 切换到规则1
/switch
# 选择规则1

# 2. 添加替换规则
/replace www.baidu.com www.google.com

# 3. 开启替换模式
/settings
# 点击开启替换模式

# 4. 切换到规则2
/switch
# 选择规则2

# 5. 重复步骤2-3
/replace www.baidu.com www.google.com
/settings
# 点击开启替换模式
```

**解决方案3：复制配置**
```bash
# 1. 先在规则2配置好所有替换规则
/switch
# 选择规则2

/replace www.baidu.com www.google.com
/replace 广告 公告
/settings  # 开启替换模式

# 2. 将规则2的配置完整复制到规则1
/copy_rule 2 1

# 完成！规则1现在有相同的替换规则了
```

### 方法3：使用Python脚本验证（推荐）
```python
# 在服务器上运行
cd /root/TelegramForwarder-Python
python3 -c "
from models.models import get_session, ForwardRule
session = get_session()
rules = session.query(ForwardRule).all()
for rule in rules:
    print(f'规则{rule.id} ({rule.source_chat.name} -> {rule.target_chat.name}):')
    print(f'  替换模式: {\"开启\" if rule.is_replace else \"关闭\"}')
    if rule.replace_rules:
        for i, rr in enumerate(rule.replace_rules, 1):
            print(f'  {i}. \"{rr.pattern}\" -> \"{rr.content}\"')
    else:
        print('  (无替换规则)')
    print()
session.close()
"
```

## ⚙️ 高级技巧

### 技巧1：使用正则表达式批量替换
```bash
# 替换所有.com域名为.net
/replace (https?://[^/]+)\.com \1.net

# 替换所有数字
/replace \d+ [数字]
```

### 技巧2：条件替换
```bash
# 只替换特定位置的文本
/replace ^广告 公告   # 只替换开头的"广告"
/replace 广告$ 公告   # 只替换结尾的"广告"
```

### 技巧3：链式替换
替换规则会按顺序执行：
```bash
/replace www.baidu.com www.google.com
/replace www.google.com www.bing.com
# 最终结果：www.baidu.com → www.bing.com
```

### 技巧4：保留部分内容
```bash
# 使用捕获组
/replace (https://.*?)/old/path \1/new/path
# 保留域名，只替换路径
```

## 🐛 故障排查

### 问题1：机器人频道有替换，目标频道没有
**原因：** 两个规则的替换规则配置不同

**解决：**
```bash
# 方案1：复制配置
/copy_rule 机器人规则ID 目标规则ID

# 方案2：批量添加
/replace_all 原文本 新文本
```

### 问题2：替换规则已添加但不生效
**检查：**
1. 替换模式是否开启？
   ```bash
   /settings
   # 查看"替换模式"开关状态
   ```

2. 规则是否正确？
   ```bash
   /list_replace
   # 检查pattern是否匹配消息内容
   ```

3. 是否在正确的规则上操作？
   ```bash
   /switch
   # 确认当前规则
   ```

### 问题3：链接替换不生效
**可能原因：**
1. 链接在Telegram中是实体，不在文本中
2. 链接格式不匹配（http vs https）

**解决：**
```bash
# 添加多个变体
/replace http://www.baidu.com https://www.google.com
/replace https://www.baidu.com https://www.google.com  
/replace www.baidu.com www.google.com
```
