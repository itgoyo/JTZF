## 基本概念

### 什么是转发规则？
转发规则定义了：
- **从哪里**（源频道/群组）
- **转发到哪里**（目标频道/群组）
- **转发什么内容**（关键字过滤）
- **如何处理内容**（文本替换）

### 核心术语
- **源频道**：消息来源的频道/群组
- **目标频道**：消息要转发到的频道/群组
- **规则ID**：每个转发规则的唯一编号
- **当前规则**：你正在操作的规则

## 快速开始

### 第一步：绑定频道

#### 方法1：基本绑定（自动使用当前聊天作为目标）
```
/bind https://t.me/source_channel
```
这会将 `source_channel` 绑定到**当前聊天窗口**

#### 方法2：指定目标频道
```
/bind https://t.me/source_channel https://t.me/target_channel
```

#### 方法3：使用频道名称（支持模糊匹配）
```
/bind "源频道名称" "目标频道名称"
```

**示例：**
```
/bind https://t.me/jiaotest0 https://t.me/jiaotest
```
将 jiaotest0 的消息转发到 jiaotest

✅ **绑定成功后**，会显示规则ID和设置按钮

---

### 第二步：添加关键字（可选）

转发规则有两种模式：
- **黑名单模式**（默认）：匹配关键字的消息**不转发**
- **白名单模式**：只转发匹配关键字的消息

#### 添加普通关键字
```
/add 关键字1 关键字2 "多个词的关键字"
```

**示例：**
```
/add 广告 推广 "点击链接"
```

#### 添加正则表达式
```
/add_regex ^广告.* .*推广.*
```

---

### 第三步：添加替换规则（可选）

替换规则会修改消息内容后再转发

#### 基本替换
```
/replace 原文本 替换后的文本
```

**示例：**
```
/replace 广告 公告
/replace www.baidu.com www.google.com
/replace https://old-link.com https://new-link.com
```

#### 删除匹配内容（留空替换内容）
```
/replace 广告
```
效果：删除消息中的"广告"二字

#### 全文替换
```
/replace .* 这是新内容
```
效果：无论原消息是什么，都替换成"这是新内容"

## 规则管理

### 查看所有规则
```
/settings
```
或
```
/settings [规则ID]
```
直接打开指定规则的设置

### 查看规则列表
```
/list_rule
```
显示所有转发规则及其ID

### 切换当前规则

当一个目标频道有多个源频道时，需要切换规则：

```
/switch
```
然后选择要操作的规则

**或者在其他命令后指定规则ID：**
```
/settings 1   # 打开规则1的设置
```

### 删除规则
```
/delete_rule 1 2 3
```
删除规则ID为1、2、3的规则

## 关键字过滤

### 查看关键字
```
/list_keyword
```

### 添加关键字
```
/add 关键字1 关键字2 "多词关键字"
```

**带引号示例：**
```
/add "这是一个广告" "点击这里"
```

### 删除关键字

#### 按内容删除
```
/remove_keyword 关键字1 "关键字2"
```

#### 按ID删除（先用/list_keyword查看ID）
```
/remove_keyword_by_id 1 3 5
```

### 批量操作

#### 添加到所有规则
```
/add_all 通用关键字
```

#### 从所有规则删除
```
/remove_all_keyword 通用关键字
```

### 清空关键字
```
/clear_all_keywords        # 清空所有关键字
/clear_all_keywords_regex  # 仅清空正则关键字
```

### 复制关键字
```
/copy_keywords 源规则ID
```
将源规则的关键字复制到当前规则

## 文本替换

### 查看替换规则
```
/list_replace
```

### 添加替换规则
```
/replace 匹配模式 替换内容
```

**常用示例：**

1. **替换链接**
```
/replace www.baidu.com www.google.com
```

2. **删除文本**
```
/replace 广告
```

3. **替换特定词语**
```
/replace 旧品牌 新品牌
```

4. **正则替换**
```
/replace https://.*\.com https://new.com
```

### 删除替换规则
```
/remove_replace 1 2 3
```
删除序号为1、2、3的替换规则

### 批量添加替换规则
```
/replace_all 广告 公告
```
为所有规则添加此替换规则

### 清空替换规则
```
/clear_all_replace
```

### 复制替换规则
```
/copy_replace 源规则ID
```

## 常见场景示例

### 场景1：转发频道A到频道B，去除广告

```bash
# 1. 绑定频道
/bind https://t.me/channelA https://t.me/channelB

# 2. 添加黑名单关键字（默认就是黑名单模式）
/add 广告 推广 "点击链接"

# 3. 完成！含有这些关键字的消息将不会转发
```

### 场景2：只转发包含特定关键字的消息

```bash
# 1. 绑定频道
/bind https://t.me/newsChannel https://t.me/myChannel

# 2. 打开设置，切换到白名单模式
/settings
# 点击按钮切换到"白名单模式"

# 3. 添加白名单关键字
/add 重要 紧急 通知

# 4. 现在只有包含"重要""紧急""通知"的消息会被转发
```

### 场景3：转发并替换链接

```bash
# 1. 绑定频道
/bind https://t.me/sourceChannel https://t.me/myChannel

# 2. 添加替换规则
/replace www.oldsite.com www.newsite.com
/replace https://ads.com/invite https://mylink.com

# 3. 替换关键词
/replace 广告 公告
/replace 旧品牌 新品牌

# 4. 完成！消息中的链接和文本都会被替换
```

### 场景4：多个源频道转发到一个目标

```bash
# 1. 绑定第一个源频道
/bind https://t.me/source1 https://t.me/target

# 2. 绑定第二个源频道到同一目标
/bind https://t.me/source2 https://t.me/target

# 3. 切换规则来管理不同的源
/switch
# 选择要管理的规则

# 4. 为每个规则单独设置关键字和替换规则
```

### 场景5：复制规则配置

假设规则1配置完美，想复制到规则2：

```bash
# 方法1：完整复制所有设置
/copy_rule 1 2

# 方法2：只复制关键字
/copy_keywords 1
/copy_keywords_regex 1

# 方法3：只复制替换规则
/copy_replace 1
```

## 导入导出

### 导出关键字
```
/export_keyword
```
生成txt文件，包含所有关键字

### 导入关键字
```
/import_keyword
```
（同时发送txt文件）

### 导出替换规则
```
/export_replace
```

### 导入替换规则
```
/import_replace
```
（同时发送txt文件）

**文件格式：**

关键字文件（每行一个）：
```
关键字1 0
关键字2 1
```
- 0 = 白名单
- 1 = 黑名单

替换规则文件（用Tab分隔）：
```
原文本1	替换文本1
原文本2	替换文本2
```

## 常见问题

### Q1: 如何知道当前在操作哪个规则？
**A:** 使用 `/switch` 查看，当前规则前面会有 ✓ 标记

### Q2: 替换规则不生效怎么办？
**A:** 检查以下几点：
1. 使用 `/list_replace` 确认规则已添加
2. 使用 `/settings` 确认"替换模式"已开启
3. 确保规则ID正确（用 `/list_rule` 查看）
4. 检查是否是对当前规则添加的替换规则

### Q3: 如何确保所有规则使用相同的配置？
**A:** 使用批量命令：
```
/add_all 通用关键字
/replace_all 原文本 新文本
```

或使用复制命令：
```
/copy_rule 源规则ID 目标规则ID
```

### Q4: 黑名单和白名单的区别？
**A:**
- **黑名单**（默认）：匹配关键字的**不转发**
- **白名单**：**只转发**匹配关键字的

在 `/settings` 中切换模式

### Q5: 正则表达式怎么写？
**A:** 常用正则：
- `^广告` - 以"广告"开头
- `广告$` - 以"广告"结尾  
- `.*广告.*` - 包含"广告"
- `https://.*\.com` - 匹配任何.com域名链接

### Q6: 如何同时转发到多个频道？
**A:** 为每个目标创建单独的规则：
```
/bind https://t.me/source https://t.me/target1
/bind https://t.me/source https://t.me/target2
```

### Q7: 转发的消息能保留原格式吗？
**A:** 
- 默认会保留原格式（链接、加粗等）
- 如果使用替换规则，只替换文本内容，格式会保持

### 💡 技巧2：测试替换规则

```

```

- 黑名单优先于白名单
- 替换规则按添加顺序执行

## 故障排查

### 消息没有被转发？
1. 检查规则是否启用：`/settings` 查看"规则状态"
2. 检查关键字是否匹配：`/list_keyword`
3. 确认转发模式（黑/白名单）

### 替换不生效？
1. 确认替换模式已开启：`/settings`
2. 查看替换规则：`/list_replace`  
3. 检查规则ID是否正确：`/list_rule`
4. 确认是否对当前规则添加了替换规则

### 不知道在操作哪个规则？
```
/switch  # 查看当前规则，有✓标记的就是
```
